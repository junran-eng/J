## 知识库智能分类

`knowledge_retriever.py` 根据文件名自动推断内容类别：

| 文件名含 | 自动分类为 |
|---------|-----------|
| 企业/概况/公司/品牌/资质 | 企业概况 |
| 技术/原理/工艺/创新 | 核心技术 |
| 案例/项目/客户/工程 | 应用案例 |
| 产品/样册/规格/参数 | 产品信息 |
| 政策/标准/法规 | 政策法规 |
| 行业/产业/市场/趋势 | 行业知识 |

## 配置方式

在 `assets/config.yaml` 中修改知识库路径：

```yaml
knowledge_base:
  path: "G:/吉康知识库/"    # 当前路径（已配置）
  backend: auto              # 自动检测（目录→文件系统后端）
  encoding: "utf-8"
  max_file_size_mb: 10
```

## 如何扩展知识库

### 添加新的知识文件

直接在 `G:/吉康知识库/` 目录下新增 `.md` 文件即可，技能会自动扫描并纳入检索范围：

```bash
# 示例：新增一个行业政策解读文件
touch "G:/吉康知识库/05-政策法规.md"
```

文件命名建议：
- 保持编号前缀 `01-` `02-` 便于排序
- 文件名含关键词便于自动分类（如 `技术`、`案例`、`政策`）
- 使用 `.md` 格式编码为 UTF-8

### 更新现有文件

直接编辑 `G:/吉康知识库/` 下的对应 `.md` 文件，技能在下次检索时会自动读取最新内容。

### 使用JSON索引（高级）

对于大规模知识库，可切换到JSON索引后端：

```bash
# 1. 生成索引文件
python3 scripts/knowledge_retriever.py --kb-path "G:/吉康知识库/" --full-content --output index.json

# 2. 修改 config.yaml
# knowledge_base:
#   path: "G:/吉康知识库/index.json"
#   backend: json
```

## 知识库检索使用

```bash
# 关键词检索
python3 scripts/knowledge_retriever.py \
  --keywords "低温除湿,热泵,能耗" \
  --kb-path "G:/吉康知识库/"

# 从内容规划中自动提取关键词检索
python3 scripts/knowledge_retriever.py \
  --plan plans/plan.json \
  --kb-path "G:/吉康知识库/" \
  --output plans/kb_results.json

# 获取完整内容（而非摘要）
python3 scripts/knowledge_retriever.py \
  --keywords "污泥,案例,经济效益" \
  --kb-path "G:/吉康知识库/" \
  --full-content
```
